# 🎯 2축 스텝모터 터렛 제어 시스템 (Turret Control System)

![Turret System Diagram](file:///c:/Users/LSK0522/OneDrive%20-%20%EC%84%9C%EC%9A%B8%EB%A1%9C%EB%B4%87%EA%B3%A0%EB%93%B1%ED%95%99%EA%B5%90/%EB%B0%94%ED%83%95%20%ED%99%94%EB%A9%B4/%EC%A1%B8%EC%97%85%EC%9E%91%ED%92%88/App_Build/turret_diagram.png)

이 프로젝트는 **카메라 비전 인식(OpenCV)** 기술과 **2축 스텝모터(DM542 드라이버)** 제어를 결합하여, 화면 상의 물체를 자동으로 추적하거나 사용자가 수동으로 조종할 수 있도록 개발된 **스마트 터렛 제어 시스템**입니다.

이 가이드는 시스템의 핵심 기능, 하드웨어/소프트웨어 아키텍처, 추적 알고리즘의 동작 방식 및 하드웨어 연동 구조를 한눈에 쉽게 파악할 수 있도록 제공합니다.

---

## 🛠️ 주요 기능 소개

### 1. 실시간 영상 스트리밍 & 수동 조종 (Manual Mode)
* **실시간 MJPEG 스트리밍:** 라즈베리 파이 카메라를 통해 들어오는 고해상도 영상을 지연시간을 최소화하여 웹 페이지에 송출합니다. (디버그 모드에서는 검출된 물체와 조준선이 실시간으로 시각화됩니다.)
* **가상 조이스틱 조작:** 웹 화면의 가상 조이스틱을 드래그하여 터렛의 Pan(수평)/Tilt(수직) 회전을 부드럽고 직관적으로 원격 제어합니다.
* **클릭 조준:** 카메라 화면 상의 임의의 위치를 클릭하거나 터치하면, 해당 방향의 위치 편차를 즉시 계산하여 그 지점으로 터렛이 자동 정렬됩니다.

### 2. 다각도 고정형 ROI 학습 및 자동 추적 (Auto Mode)
* **동적 ROI 드래그 & 칸 고정:** 사용자가 화면에 직접 사각형 박스를 드래그하여 지정하면 해당 범위가 화면에 고정되어 안정적인 학습 범위를 확보합니다.
* **실시간 진행률 피드백:** 학습이 실행되면 화면에 퍼센트(%) 게이지 바가 차오르는 실시간 진행 상태(0~100%)가 시각화되어 정상 학습 여부를 즉각 확인할 수 있습니다.
* **다각도 지문인식형 반복 학습:** 대상을 카메라 앞에서 돌려가며 추가 학습(`Add Learning`)을 시킬 때마다 여러 각도의 특징점을 지문인식 방식처럼 누적시켜 검출 정밀도를 비약적으로 증대시킵니다.
* **실시간 자동 추적:** 학습이 끝난 타겟이 움직이면 특징점을 추적하여 터렛이 대상물과 조준선을 항상 일치시킵니다.

### 3. 비상 탈출 및 세션 취소
* **ESC 키 전역 세션 탈출:** ROI 드래그 영역 지정 중이거나, 학습 게이지가 차오르는 중간 단계 등 언제든 키보드의 `Escape` 키를 누르면 진행 중이던 학습 세션이 즉시 폐기되고 안전하게 조작 상태로 복귀합니다.

### 4. 하드웨어 세부 튜닝 및 상태 모니터링
* **실시간 파라미터 튜닝:** 픽셀당 이동 스텝 수, 데드존(허용 오차 범위), 모터 구동 속도 및 감지 민감도 등의 정밀 매개변수를 슬라이더 UI를 통해 웹 화면에서 즉각 동적으로 수정 및 적용할 수 있습니다.
* **실시간 피드백:** 터렛의 오차 픽셀 수, 출력되고 있는 스텝 수, 연결 포트 정보, 통신 타임아웃 상태를 실시간 데이터로 확인할 수 있습니다.
* **하드웨어 커넥션 탐지:** ESP32 또는 아두이노 우노가 장착되어 신호를 주고받는 경우, 연결 상태를 실시간 감지하여 상태 인디케이터에 표시합니다. 연결이 끊기거나 누락될 경우 즉각적인 경고가 화면에 노출됩니다.

### 5. 미디어 라이브러리 및 갤러리
* **카메라 스냅샷:** 셔터 버튼을 클릭하는 순간 화려한 플래시 효과와 함께 현재 프레임을 이미지 파일로 캡처합니다.
* **모던 이미지 갤러리:** 저장된 이미지를 슬라이드 형식으로 탐색하며 개별로 로컬 기기에 다운로드할 수 있습니다.

## 🧠 스마트 객체 추적 알고리즘의 원리

다양한 주변 환경과 노이즈에서도 흔들림 없고 끈질긴 추적 감도를 보장하기 위해 **5대 알고리즘(YCrCb 스킨 마스크, 차분 모션 마스크, ORB 특징점 매칭, Hough Circle 기하학 검출, Kalman Filter 추적)**을 하나의 파이프라인으로 구성했습니다.

```mermaid
graph TD
    Frame["카메라 프레임 입력"] --> Diff["차분 모션 마스크<br>(움직이는 영역 제한)"]
    Frame --> Skin["YCrCb 피부색 마스크<br>(사람 손/피부 영역 차단)"]
    
    Diff & Skin --> ORB{"1. ORB 특징점 매칭<br>(일치하는 특징점 충분?)"}
    
    ORB -->|Yes (인식 성공)| Kalman["칼만 필터 업데이트<br>(좌표 상태 및 예측값 갱신)"]
    ORB -->|No (특징점 부족)| Hough{"2. Hough Circle 폴백<br>(원형 외형 검출?)"}
    
    Hough -->|Yes (원형 검출)| Kalman
    Hough -->|No (검출 실패)| KalmanPredict{"3. 칼만 필터 예측<br>(최대 20프레임 관성 추적)"}
    
    KalmanPredict -->|예측 성공| Motor["조준점 업데이트 ➡️ 모터 동작"]
    KalmanPredict -->|예측 타임아웃| Lost["추적 정보 소실 (Target Lost)"]
    
    Kalman --> Motor
```

### 1. 피부색(Skin) 및 모션(Motion) 차단 필터링
* **YCrCb 스킨 컬러 마스크:** 조도 불균형에 강인한 `YCrCb` 색상 공간으로 변환한 뒤, 피부색 범위(`Cr: 133~173, Cb: 77~127`)를 지정하여 추적 대상에서 아예 누락시킵니다. 이 기술은 **사용자가 대상을 잡고 조종할 때 손가락을 표적으로 오검출하지 않도록 보장**합니다.
* **차분 모션 필터 (Frame Difference):** 현재 프레임과 이전 프레임 간의 명암 변화를 감지하여 동적 객체가 존재하는 영역에 연산을 집중시킵니다. 정적인 벽지 무늬나 가구 등 뒷배경의 복잡한 특징점을 노이즈로 오인하는 것을 방지합니다.

### 2. ORB(Oriented FAST and Rotated BRIEF) 특징점 정합
* **특징점 학습 및 저장:** 사용자가 드래그 지정한 ROI 영역에서 피부색과 정적 배경을 제외하고 순수한 물체 표면의 무늬/모서리 정보를 추출해 descriptors 형태로 학습 파일에 영구 저장합니다.
* **FLANN 고속 매칭:** 획득 중인 실시간 카메라 영상의 특징점 풀과 미리 저장한 학습 데이터베이스를 빠른 속도로 크로스 체크하여 최적의 부합 지점을 찾아내고 타겟 중심 좌표를 계산합니다.

### 3. 허프 원 변환(Hough Circle) 폴백 시스템
* 단색의 당구공이나 완전 백색의 탁구공 등 **표면 질감과 고유 무늬가 부재한 물체**는 ORB로 분석할 특징점이 없어 검출을 실패하기 쉽습니다.
* 이를 보완하기 위해 ORB 인식을 실패하면 형태학적 기하 구조로 원을 찾는 **허프 원 변환**이 자동으로 구동되어 보완합니다.

### 4. 칼만 필터(Kalman Filter)를 통한 관성 예측
* 물체가 극도로 빠르게 이동하거나, 카메라 프레임 바깥으로 일시 이탈하거나, 장애물에 가려지는 상태(Occlusion)가 발생할 때 추적이 뚝뚝 끊기는 문제를 개선합니다.
* 2축 속도 및 가속도를 바탕으로 움직임 방정식을 갱신하여 **물체가 사라진 순간에도 최대 20프레임 동안 예상 궤적으로 터렛을 관성 이동**시킴으로써 물체 재포착 속도를 극대화합니다.

## ⚡ 하드웨어 및 펌웨어 제어 원리

### 1. ESP32 듀얼코어 및 FreeRTOS 병렬 아키텍처
스텝 모터의 구동을 위한 펄스(`PUL`) 출력은 마이크로초(μs) 수준의 미세 지연 연산(`delayMicroseconds`)을 소모합니다. 만약 단일 루프에서 시리얼 통신와 모터 펄스 출력을 같이 처리하면 통신 지연이나 스텝 누락이 생깁니다.
이 문제를 원천 방지하기 위해 [motor_esp32.py](file:///c:/Users/LSK0522/OneDrive%20-%20%EC%84%9C%EC%9A%B8%EB%A1%9C%EB%B4%87%EA%B3%A0%EB%93%B1%ED%95%99%EA%B5%90/%EB%B0%94%ED%83%95%20%ED%99%94%EB%A9%B4/%EC%A1%B8%EC%97%85%EC%9E%91%ED%92%88/App_Build/turret_ws/motor_esp32.py) 및 ESP32 전용 펌웨어는 듀얼코어를 활용하여 두 가지 독립 작업을 병렬로 처리합니다.

* **Core 0 (`serialTask`):** 시리얼 수신 포트를 백그라운드에서 실시간 모니터링하여 목표 좌표 명령(`T:320:240`)을 입수하는 즉시 디코딩하여 메모리 값을 업데이트합니다.
* **Core 1 (`motorTask`):** 10ms 주기 타이머에 맞춰 연동되며, 목표 편차 값에 따라 모터 드라이버로 물리 펄스를 직접 발생시킵니다.
* 두 태스크는 세마포어(`Semaphore/Mutex`)를 통과하여 공유 리소스의 메모리 충돌을 완전 차단합니다.

### 2. 거리 오차 기반 모터 비례 제어 (Proportional Control)
PID 제어 중 비례항(P) 원리를 적용하여 조준 정확도를 개선했습니다.
* 조준 좌표와 화면 중심 좌표 간의 픽셀 거리 편차 값에 맞춰 출력할 회전 스텝량을 유동적으로 변형합니다.
* **제어 수식:**
  $$\text{Steps} = \text{constrain}(\text{abs(Error)} \times \text{steps\_per\_px}, 1, \text{max\_steps})$$
* **작동 프로세스:**
  * **대형 오차 (목표가 멀 때):** 편차 값이 큰 범위에서는 출력 펄스 수를 한계치(`max_steps`)까지 밀어붙여 **빠르고 강력하게 표적의 방향으로 선회**합니다.
  * **미세 오차 (근접할 때):** 타겟이 조준선에 바짝 가까워질수록 펄스 발생 수를 1~2스텝 수준으로 잘게 쪼개어 속도를 늦춤으로써 **오버슈트(Overshoot)와 떨림이 전혀 없는 정지 동작**을 구축합니다.
  * **데드존(Dead-Zone) 제동:** 편차 거리가 데드존 크기(기본: 8픽셀) 안으로 진입하는 순간 펄스 동작을 정지시켜 노이즈에 의한 모터의 미세 떨림과 마모를 차단합니다.

## 📊 디바이스 통신 모드 비교

사용자가 설정 메뉴에서 선택한 컨트롤러 유형에 맞게 통신 프로토콜과 구동 체계가 완전히 이원화되어 작동합니다.

| 비교 구분 | 🟢 ESP32 제어 방식 (`esp32`) | 🔵 Arduino Uno 제어 방식 (`arduino`) |
| :--- | :--- | :--- |
| **통신 패킷 포맷** | 단문 텍스트 스트림 (`T:x:y\n`, `CFG:K:V\n`) | 메모리가 제한된 환경에 적합한 초경량 JSON 인코딩 패킷 |
| **제어 연산의 주체** | 라즈베리 파이는 표적의 픽셀 좌표만 전송하고, **ESP32 자체**에서 오차 비례 및 주파수 변환 연산 진행 | **라즈베리 파이 백엔드**에서 타겟 오차를 연산하여 환산된 스텝 이동 명령을 직접 밀어 넣는 방식 |
| **모터 반응성** | 매 프레임 수신되는 좌표 데이터 스트림을 타고 **연속적이고 매끄러운 트랙킹 연동** 실현 | 명령을 받아 특정 위치까지 회전하고 완료 응답을 보내는 **이벤트형 동기식 정밀 스텝 구동**에 강함 |
| **주요 활용 추천** | 실시간 원격 물체 추적 제어 및 라이브 조이스틱 운용 | 단순 포지셔닝 실험, 센서 보정, 정밀 스텝 이동용 테스트 |

## 📖 사용 방법 및 타겟 학습 가이드

> [!TIP]
> **대상을 가장 안정적으로 감지하고 추적시키기 위한 학습 노하우**
> 1. 웹 설정 패널에서 **'물건 학습하기'** 모드를 활성화합니다.
> 2. 마우스를 드래그하여 **학습시킬 물체 크기에 거의 딱 들어맞는 푸른색 ROI 상자**를 그립니다. 주변 배경이 지나치게 많이 침투하면 오인 학습의 원인이 되므로, 표적 영역 크기에 최대한 맞추어 그려주는 것이 핵심입니다.
> 3. **'이 영역 학습 시작'**을 클릭하면 학습 중임을 알리는 녹색 프레임으로 상태가 전환되며 **4초의 타이머**가 작동하고 % 게이지 바가 차오르기 시작합니다.
> 4. 이 4초간 카메라 바로 앞에서 물체를 **천천히 360도 회전**시키며 앞면, 옆면, 뒷면의 무늬가 골고루 카메라에 노출되도록 회전시킵니다.
> 5. 1차 수집이 끝나면 썸네일과 함께 추가 진행 여부를 묻는 창이 발생합니다. 이때 **'추가 학습(Add Learning)'**을 눌러 카메라와의 거리를 조절(앞으로 당기거나 멀리 보냄)한 상태에서 2~3회 반복 학습해주면 인식 감도와 거리에 따른 오검출을 극적으로 개선할 수 있습니다.
> 6. 모든 반복 수집을 마친 후 **'학습 종료'**를 누르고, 모드를 **'자동 조종'**으로 변경하면 터렛이 타겟의 고유 특징을 바탕으로 정밀 추적을 개시합니다.

> [!WARNING]
> **학습 도중 취소(ESC) 및 특징점 경고 안내**
> * 학습 도중에 물체가 너무 가깝거나 너무 어두워서 특징점(Descriptor)이 0개로 검출될 경우, 상태창에 **'학습 실패: 특징점을 감지할 수 없습니다. 조명을 밝게 하거나 물체 위치를 조정해주세요.'** 라는 빨간 경고 문구가 노출됩니다.
> * 드래그 중이거나 학습 바가 차오르는 중간에 물체 배치를 수정하고 싶다면 키보드의 **`Escape(ESC)` 키**를 터치하여 세션을 즉시 폭파하고 언제든 원점으로 되돌아갈 수 있습니다.
